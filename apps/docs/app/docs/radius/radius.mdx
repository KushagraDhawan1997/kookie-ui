# Radius

Border radius system with 5-step scale for consistent corner rounding. Kookie UI Radius is extracted from [Radix Themes](https://www.radix-ui.com/themes) with refined values for better visual hierarchy. 

For complete radius documentation and advanced usage, visit the [Radix Themes radius docs](https://www.radix-ui.com/themes/docs/theme/radius). Note that Kookie UI's radius values may be enhanced and might not match exactly what's shown in the Radix documentation.

</Section>

<Section size="1">

## Usage

Set `radius` prop to control border radius across all components. Use consistent radius values to create visual hierarchy and polish.

<CodeBlock>

```tsx
import { Theme } from '@kushagradhawan/kookie-ui';
```

</CodeBlock>

<CodeBlock>

```tsx
<Theme radius="medium">
  <YourApp />
</Theme>
```

</CodeBlock>

## Radius Factor System

Theme component uses radius factors that multiply base radius values:

<CodeBlock>

```css
/* Radius factor system */
[data-radius='none']   { --radius-factor: 0; }     /* 0x multiplier */
[data-radius='small']  { --radius-factor: 0.75; }  /* 0.75x multiplier */
[data-radius='medium'] { --radius-factor: 1; }     /* 1x multiplier */
[data-radius='large']  { --radius-factor: 1.5; }   /* 1.5x multiplier */
[data-radius='full']   { --radius-factor: 2.0; }   /* 2x multiplier */
```

</CodeBlock>

<CodeBlock>

```tsx
/* Theme radius usage */
<Theme radius="none">    /* All radius values become 0px */
<Theme radius="small">   /* All radius values × 0.75 */
<Theme radius="medium">  /* All radius values × 1 (default) */
<Theme radius="large">   /* All radius values × 1.5 */
<Theme radius="full">    /* All radius values × 2 */
```

</CodeBlock>

## Differences from Radix

Kookie UI refines the Radix Themes radius system with more granular control and better visual hierarchy:

**Refined Values:**
Smaller increments (2px, 4px, 6px, 8px, 12px) for more precise control. Better proportions where each step provides meaningful visual difference and consistent scaling for more predictable progression across the scale.

**Enhanced Usage:**
Component-specific defaults where each component uses the most appropriate radius. Semantic naming with clear, descriptive radius names and better accessibility with radius values that work well with focus indicators.
