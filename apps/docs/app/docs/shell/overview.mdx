# Shell

The Shell component is a complete layout engine that provides structural patterns for building application interfaces. It manages layout state, composition rules, and responsive behavior across seven core slots: Header, Rail, Panel, Sidebar, Content, Inspector, and Bottom.

## [Unreleased]

### Added
- Complete Shell component implementation with seven core slots
- Responsive presentation modes: fixed, overlay, and stacked
- Advanced resize system with keyboard navigation and accessibility
- Composition rule enforcement (Rail+Panel vs Sidebar exclusivity)
- Comprehensive breakpoint system with mobile-first approach
- Size persistence API for maintaining user preferences
- Peek functionality for temporary pane previews
- `useShell()` hook for accessing shell context and control functions
- `useResponsivePresentation()` hook for resolving responsive presentation values
- ARIA slider implementation for resize handles
- Semantic HTML structure with proper landmarks
- Focus management for overlay/stacked presentations
- Screen reader announcements for state changes

### Technical
- Built on modern React patterns with context splitting for performance
- Comprehensive TypeScript types for all configurations
- Six breakpoints: initial, xs, sm, md, lg, xl
- Mobile-first approach with overlay defaults
- Automatic breakpoint detection and state resolution

## [0.1.41] - 2024-01-15

### Added
- Initial beta release of Shell component
- `Shell.Root` - Layout container with height control
- `Shell.Header` - Fixed header with customizable height
- `Shell.Rail` - Narrow navigation strip (64px default)
- `Shell.Panel` - Expandable sidebar panel (288px default)
- `Shell.Sidebar` - Alternative unified navigation with thin/expanded modes
- `Shell.Content` - Main content area (always required)
- `Shell.Inspector` - Right-side contextual panel (320px default)
- `Shell.Bottom` - Bottom panel for terminals/logs (200px default)
- `Shell.Trigger` - Button triggers with peek-on-hover support
- Size persistence adapter interface
- Keyboard navigation with arrow keys, Home/End
- Responsive mode and presentation configurations

### Technical
- Context splitting for optimal performance
- TypeScript support with comprehensive type definitions
- Design token integration for consistent spacing and sizing
- CSS custom properties for theming
