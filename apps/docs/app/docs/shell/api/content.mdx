# API

The Shell component provides a comprehensive layout system for building application interfaces with collapsible panels, responsive behavior, and flexible composition patterns.

</Section>

<Section size="1">

## Props

<CodeBlock buttonsPosition="center">

```tsx
import { Shell } from '@kushagradhawan/kookie-ui';
```

</CodeBlock>

<CodeBlock>

```tsx
<Shell.Root>
  <Shell.Header>
    {/* Global navigation */}
  </Shell.Header>
  
  <Shell.Rail>
    {/* Navigation rail */}
  </Shell.Rail>
  
  <Shell.Panel>
    {/* Sidebar panel */}
  </Shell.Panel>
  
  <Shell.Content>
    {/* Main content area */}
  </Shell.Content>
  
  <Shell.Inspector>
    {/* Right inspector panel */}
  </Shell.Inspector>
  
  <Shell.Bottom>
    {/* Bottom panel */}
  </Shell.Bottom>
</Shell.Root>
```

</CodeBlock>

</Section>

## Composition Rules

<Callout.Root variant="soft" size="2" color="orange" my="6" highContrast><Callout.Text>The Shell component enforces specific composition rules to maintain layout consistency and prevent conflicting arrangements.</Callout.Text></Callout.Root>

### Valid Combinations

**Rail + Panel**: These components work together. When Rail collapses, Panel automatically collapses.

<CodeBlock>

```tsx
<Shell.Root>
  <Shell.Rail>...</Shell.Rail>
  <Shell.Panel>...</Shell.Panel>
  <Shell.Content>...</Shell.Content>
</Shell.Root>
```

</CodeBlock>

**Sidebar Only**: Sidebar cannot coexist with Rail or Panel.

<CodeBlock>

```tsx
<Shell.Root>
  <Shell.Sidebar>...</Shell.Sidebar>
  <Shell.Content>...</Shell.Content>
</Shell.Root>
```

</CodeBlock>

### Invalid Combinations

<Callout.Root variant="soft" size="2" color="red" highContrast my="6"><Callout.Text>The following combinations will trigger console warnings and may cause layout conflicts.</Callout.Text></Callout.Root>

<CodeBlock>

```tsx
// ❌ Invalid: Sidebar with Rail/Panel
<Shell.Root>
  <Shell.Sidebar>...</Shell.Sidebar>
  <Shell.Rail>...</Shell.Rail>
  <Shell.Panel>...</Shell.Panel>
  <Shell.Content>...</Shell.Content>
</Shell.Root>
```

## Accessibility

The Shell component provides comprehensive accessibility support through semantic HTML structure, ARIA attributes, and keyboard navigation patterns.

### Semantic Structure

The Shell uses proper landmark elements and ARIA roles:
- `Shell.Root` - Main application container
- `Shell.Header` - Header element for global navigation
- `Shell.Rail` - Navigation container
- `Shell.Panel` - Sidebar container
- `Shell.Sidebar` - Alternative navigation container
- `Shell.Content` - Main landmark for primary content
- `Shell.Inspector` - Right panel container
- `Shell.Bottom` - Bottom panel container

### Keyboard Navigation

**Panel Controls:**
- Tab navigation through all interactive elements
- Escape key to close overlays and return focus
- Arrow keys for navigation within panels
- Enter/Space for activation

### Screen Reader Support

**State Announcements:**
- Panel open/close states
- Resize operations with current dimensions
- Responsive mode changes
- Focus management during overlays

**ARIA Attributes:**
- `aria-expanded` for collapsible panels
- `aria-hidden` for hidden content
- `aria-label` for resize handles
- `aria-describedby` for contextual information

### Focus Management

**Overlay Presentations:**
- Focus trapping within modal overlays
- Return focus to trigger element on close
- Logical tab order maintenance
- Escape key handling for dismissal

**Responsive Behavior:**
- Focus management during layout changes
- Proper landmark navigation
- Screen reader announcements for mode changes

### High Contrast Support

- Enhanced focus indicators for all interactive elements
- Proper contrast ratios for resize handles
- Clear visual feedback for all states
