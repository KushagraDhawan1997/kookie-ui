## Elevation

Shadows aren't just visual effectsâ€”they're tools for creating elevation and hierarchy. In interface design, elevation determines what feels important, what's interactive, and how users understand the spatial relationships between elements.

The shadow system is built around elevation principles:

- **Progressive elevation** - Each step adds meaningful depth without being overwhelming
- **Consistent opacity** - All shadows use the same base opacity for visual consistency
- **Purposeful blur** - Blur radius increases with elevation to create natural depth
- **Subtle offsets** - Y-offsets are minimal to avoid harsh drop shadows

Elevation creates visual hierarchy. Elements that are elevated feel more important and interactive. The right shadow level can make a button feel clickable, a card feel substantial, or a dialog feel like it's floating above everything else.

## Usage

Shadow tokens can be accessed using CSS variables. Use these tokens to style your custom components, ensuring they're consistent with the rest of the design system.

<CodeBlock>

```css
/* Inset shadow for pressed states */
.my-component {
  box-shadow: var(--shadow-1);
}

/* Card with subtle elevation */
.my-card {
  box-shadow: var(--shadow-2);
}

/* Elevated button */
.my-button {
  box-shadow: var(--shadow-3);
}

/* Overlay panel */
.my-overlay {
  box-shadow: var(--shadow-5);
}

/* Modal dialog */
.my-dialog {
  box-shadow: var(--shadow-6);
}
```
